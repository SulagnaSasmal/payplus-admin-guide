# PayPlus Enterprise v3.2 — Configuration Runbook

This runbook is a complete reference for PayPlus Enterprise configuration. It covers environment variables, feature flags, connection string templates, TLS configuration, and integration endpoint settings. Use it alongside the [deployment guide](deployment-guide.md) during installation and as a reference during operations.

**Audience:** System administrators and infrastructure engineers who configure and maintain PayPlus Enterprise environments.

---

## Configuration file locations

| File | Purpose |
|---|---|
| `conf/payplus.properties` | Primary application configuration |
| `conf/payplus-install.properties` | Installation-time parameters (read once at install; not used at runtime) |
| `conf/log4j2.xml` | Logging configuration |
| `conf/keystore.jks` | TLS certificate store (auto-generated at install; replace for production) |
| `conf/payment-rails.properties` | Payment rail connector settings (ACH, Fedwire, SWIFT, RTP, FedNow) |
| `conf/ofac.properties` | OFAC sanctions screening configuration |

After changing any property file, restart PayPlus for the change to take effect:

```bash
./bin/payplus-stop.sh && ./bin/payplus-start.sh
```

## Environment variables

PayPlus reads the following environment variables at startup. Set them in the service configuration or the shell that runs the startup script. Values in `conf/payplus.properties` override environment variables.

| Variable | Required | Default | Description |
|---|---|---|---|
| `PAYPLUS_DB_PASSWORD` | Yes | — | Database password for `PAYPLUS_APP`. Do not store the password in `payplus.properties` — use this variable instead. |
| `PAYPLUS_LICENSE_KEY` | No | — | Inline license key, if not using a license file. Set one or the other, not both. |
| `PAYPLUS_LOG_LEVEL` | No | `INFO` | Root log level. Accepted values: `TRACE`, `DEBUG`, `INFO`, `WARN`, `ERROR`. |
| `PAYPLUS_JVM_OPTS` | No | `-Xms4g -Xmx12g` | Additional JVM flags. Do not change heap sizes here if `app.jvm.heap.min` / `app.jvm.heap.max` are set in `payplus.properties`. |
| `PAYPLUS_HTTPS_PORT` | No | `8443` | HTTPS port for the administration console. |
| `PAYPLUS_SMTP_PASSWORD` | No | — | SMTP server password for notification emails. |

Example — set before starting PayPlus:

```bash
export PAYPLUS_DB_PASSWORD="$(cat /run/secrets/payplus_db_password)"
export PAYPLUS_LOG_LEVEL="WARN"
./bin/payplus-start.sh
```

## Core configuration parameters (`conf/payplus.properties`)

### Database connection

| Parameter | Required | Example | Description |
|---|---|---|---|
| `db.type` | Yes | `oracle` | Database type. Accepted values: `oracle`, `sqlserver`. |
| `db.host` | Yes | `db-server.internal` | Database hostname or IP. For Oracle RAC, use the SCAN listener address. |
| `db.port` | Yes | `1521` | Database port. Oracle default: `1521`. SQL Server default: `1433`. |
| `db.service_name` | Oracle | `PAYPLUS` | Oracle service name. Do not use `db.service_name` and `db.sid` together. |
| `db.sid` | Oracle (deprecated) | `PAYPLUS` | Oracle SID. Use `db.service_name` for Oracle 12c and later. |
| `db.database_name` | SQL Server | `PAYPLUS` | SQL Server database name. |
| `db.username` | Yes | `PAYPLUS_APP` | Database user. |
| `db.pool.min` | No | `5` | Minimum connection pool size. |
| `db.pool.max` | No | `50` | Maximum connection pool size. Each connection uses approximately 10 MB of RAM. |
| `db.pool.timeout_ms` | No | `30000` | Maximum milliseconds to wait for a pooled connection before throwing an error. |
| `db.pool.validation_query` | No | `SELECT 1 FROM DUAL` | SQL query to validate idle connections. Change to `SELECT 1` for SQL Server. |

### Connection string templates

**Oracle (service name):**

```
jdbc:oracle:thin:@//db-server.internal:1521/PAYPLUS
```

**Oracle (RAC — SCAN listener):**

```
jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=scan-listener.internal)(PORT=1521)))(CONNECT_DATA=(SERVICE_NAME=PAYPLUS_SVC)))
```

**SQL Server:**

```
jdbc:sqlserver://db-server.internal:1433;databaseName=PAYPLUS;encrypt=true;trustServerCertificate=false
```

> **Note:** For SQL Server with Windows Authentication (Kerberos), add `;integratedSecurity=true` and include the `mssql-jdbc_auth-*.dll` in the library path. Contact Microsoft for the DLL.

### Application server

| Parameter | Required | Default | Description |
|---|---|---|---|
| `app.hostname` | Yes | — | The fully qualified hostname users and services use to reach this server. Used in URLs generated by the application. |
| `app.https.port` | No | `8443` | HTTPS port for the administration console. |
| `app.session.timeout_minutes` | No | `30` | User session idle timeout. Sessions expire silently — users must re-authenticate. |
| `app.max_file_upload_mb` | No | `50` | Maximum size for bulk payment file uploads (ACH, SWIFT MT files). |
| `app.jvm.heap.min` | No | `4g` | JVM minimum heap. Must be less than or equal to `app.jvm.heap.max`. |
| `app.jvm.heap.max` | No | `12g` | JVM maximum heap. Do not exceed 75% of available RAM. |

## Feature flags

Feature flags enable or disable non-default functionality. All flags are boolean (`true` / `false`).

| Flag | Default | Description |
|---|---|---|
| `feature.dual_control.enabled` | `false` | Requires two-user approval for all payment actions above the threshold in `workflow.dual_control.threshold_usd`. Enable for regulated environments. |
| `feature.ofac_screening.enabled` | `true` | Enables real-time OFAC sanctions screening on all payment submissions. Disable only in fully isolated sandbox environments. |
| `feature.sso.enabled` | `false` | Enables SAML 2.0 single sign-on. Requires `sso.*` parameters and an identity provider configuration. |
| `feature.audit_log.immutable` | `false` | Writes audit log entries to an append-only table that cannot be modified or deleted by application users. Enable for SOX/PCI environments. |
| `feature.payment_templates.enabled` | `true` | Allows users to save recurring payment configurations as reusable templates. |
| `feature.bulk_ach.enabled` | `true` | Enables bulk ACH file upload and batch processing. Disable if your license does not include batch ACH. |
| `feature.fednow.enabled` | `false` | Enables FedNow instant payment connector. Requires a FedNow participant account and a completed `conf/payment-rails.properties` FedNow section. |
| `feature.rtp.enabled` | `false` | Enables RTP (The Clearing House) instant payment connector. |

## TLS/SSL configuration

PayPlus generates a self-signed certificate during installation. Replace it with a CA-signed certificate before production use.

### Replace the keystore

1. Obtain a JKS-format keystore containing your certificate and private key. If you have a PEM certificate and key, convert with keytool:

   ```bash
   # Step 1: Create a PKCS12 bundle from the PEM cert and key
   openssl pkcs12 -export \
     -in /path/to/cert.pem \
     -inkey /path/to/key.pem \
     -certfile /path/to/ca-chain.pem \
     -out payplus.p12 \
     -name payplus \
     -passout pass:KEYSTORE_PASSWORD

   # Step 2: Import into a JKS keystore
   keytool -importkeystore \
     -srckeystore payplus.p12 \
     -srcstoretype PKCS12 \
     -srcstorepass KEYSTORE_PASSWORD \
     -destkeystore conf/keystore.jks \
     -deststoretype JKS \
     -deststorepass KEYSTORE_PASSWORD \
     -alias payplus
   ```

2. Update `conf/payplus.properties`:

   ```properties
   tls.keystore.path=conf/keystore.jks
   tls.keystore.password=KEYSTORE_PASSWORD
   tls.key.alias=payplus
   tls.protocols=TLSv1.2,TLSv1.3
   tls.ciphers=TLS_AES_256_GCM_SHA384,TLS_CHACHA20_POLY1305_SHA256,TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
   ```

3. Restart PayPlus and verify the certificate in a browser.

> **Do not** use TLS 1.0 or 1.1. NACHA and PCI DSS 4.0 prohibit these versions for systems that handle payment data.

## Integration endpoint settings

### ACH gateway

```properties
ach.gateway.host=ach.paymentsnetwork.internal
ach.gateway.port=4443
ach.gateway.routing_number=021000021
ach.gateway.company_id=YOUR_COMPANY_ID
ach.gateway.effective_date_offset=1
ach.gateway.cutoff_time=17:00:00-05:00
```

### SMTP (notifications)

```properties
smtp.host=smtp.internal
smtp.port=587
smtp.starttls=true
smtp.username=payplus-notify@example.com
smtp.from_address=payplus-notify@example.com
smtp.from_name=PayPlus Enterprise
```

The SMTP password comes from the `PAYPLUS_SMTP_PASSWORD` environment variable, not from `payplus.properties`.

## Logging configuration (`conf/log4j2.xml`)

The default configuration writes to `logs/payplus.log` with daily rotation and a 30-day retention policy. To increase verbosity for a specific component without enabling full DEBUG mode:

```xml
<!-- Add inside <Loggers> -->
<Logger name="com.payplus.ach" level="DEBUG" additivity="false">
  <AppenderRef ref="RollingFile"/>
</Logger>
```

Restart PayPlus after changing `log4j2.xml`. Log level changes take effect without restart only if you set `monitorInterval="60"` on the `<Configuration>` element:

```xml
<Configuration monitorInterval="60">
```
